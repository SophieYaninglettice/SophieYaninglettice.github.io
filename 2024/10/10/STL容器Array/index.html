STL标准容器库 array 的一个简单实现

为什么引入std::array?

引入std::array，而非使用原生数组有以下原因：

  1. 与原生数组相比，std::array拥有更好的类型安全和边界检查
  2. std::array是对原生数组进行了封装，拥有新的特性的同时更加安全和现代化
  3. 支持所有标准库特性，拥有和原生数组接近的效率，在大部分情况可以完全取代原生数组。

原生数组 VS std::array 类型安全与边界检查

原生数组

  • 原生数组只提供了最基础的类型检查，原生数组在声明时指定类型，并在编译时进行类型检查。例如：
  • 某些情况下，原生数组会退化为指针，这可能导致类型安全性问题
  • 原生数组没有在对边界检查提供任何内置支持，如数组越界访问，开发者需要手动进行边界检查

std::array

  • array 容器是一个模板类，具有两个模板参数：数组数据类型和数据长度。在编译器进行模板实例化，实例化后，编译器会进行类型检查
  • array容器提供了内置的机制和工具进行边界检查
  • array容器作为STL容器，支持标准迭代器接口，可以通过提供的迭代器接口进行访问，从而避免显式地访问导致的索引越界问题
  • array容器支持调试模式，可以开启调试模式启用额外的允许时检查

原生数组 VS std::array 内存布局

原生数组

  • 原生数组在内存上是一个连续的内存地址

std::array

  • std::array 在本质上其实是对原生数组作了一层封装,内部存储数据的data部分本质上就是原生数组

原生数组 VS std::array 对于C++ 新特性的支持及自身特性

原生数组

  • 作为对C语言的继承和扩充，C++ 对原生数组的扩展比较少，原生数组更多作为一种类似 “基础类型” 而存在（基础类型的初级扩展）

std::array

  • std::array 继承了原生数组的几乎全部特性（除动态分配外），并在此基础上扩充了新的特性。

在硬件资源支持的情况下，使用标准容器库能让C++开发更加现代化。

  • 有关拷贝（拷贝语义）

在C++中，拷贝是指对象的内容被复制到另一个对象中。这可以通过多种方式实现，主要实现方式包括拷贝构造函数和拷贝赋值运算符。在未定义拷贝构造函数和拷贝赋值运算符时，编译器会生成默认的拷贝构造函数和拷贝赋值运算符，一般来说，编译器默认的拷贝行为都是浅拷贝（Shallow Copy）

  • 浅拷贝：仅复制对象的值，包括指针的值。复制指针的值的含义是：将一个指针变量所存储地址值赋值给另一个指针变量，从“指向”的角度而言，这并未创建新的内容，而是同时有了两个指针指向了相同的内存块。在需要进行动态内存分配和管理的场合，这通常会引起问题。

  • 深拷贝：不仅复制对象的值，还会分配新的内存并复制指针所指向的实际内容。

  • Rule of three : 三法则，C++认为，如果一个类需要自定义析构函数、拷贝构造函数或拷贝赋值运算符中的任何一个，那么它通常需要这三个。（后在C++11中得到了扩充，Rule of five: 五法则，如果一个类需要自定义析构函数、拷贝构造函数、拷贝赋值运算符、移动构造函数或移动赋值运算符中的任何一个，那么它通常需要这五个。这两个法则通常被称为“三五法则”）

  • Rule of zero ：零规则，具有自定义析构函数、复制/移动构造函数或复制/移动赋值运算符的类应专门处理所有权。其他类不应具有自定义析构函数、复制/移动构造函数或复制/移动赋值运算符。与RAII(Resource Acquisition Is Initialization，资源获取即初始化)相关，本文不作过多介绍。

然后，需要对拷贝赋值运算符进行重载

  • 有关移动（移动语义）：移动语义是 C++11 引入的重要特性，旨在提高程序的性能和效率，特别是在处理临时对象和资源管理时。c++11在三法则的基础上扩充了五法则，新增的部分是移动语义相关的移动构造函数和移动赋值运算符。

  • 左值（Lvalues）: 具有名称或者可以取地址的表达式。通常是持久性的，例如变量或者函数返回的引用。

  • 右值（Rvalues）：没有名称或者不能取地址的表达式。通常是临时性的，例如字面常量、临时对象或者函数返回的临时对象。

  • 移动构造函数 ：允许将资源从一个临时对象（右值）”窃取” 到新对象中，而不是复制。移动构造函数通常使用 && 引用来接受右值引用参数。

  • 移动赋值运算符 : 用于从一个临时对象（右值）”窃取” 资源到另一个对象中，而不是进行复制。通常也使用 && 引用来接受右值引用参数。

  • “窃取” ：指的是从一个临时对象或者右值对象获取资源（如堆上分配的内存、文件句柄等），而不是进行深度复制。通过移动语义，可以高效地将资源所有权从一个对象转移给另一个对象，避免不必要的资源复制和释放开销。

  • 析构函数

析构函数在对象生命周期结束时调用，用于回收对象所占用的资源。静态数组的内存是由编译器在对象生命周期结束时自动管理和释放的，因此使用默认的析构函数即可。

理解迭代器

模板使得算法独立于存储的数据类型，而迭代器使得算法独立于使用的数据容器。

模板(基础补充)

模板是一种让函数和类能够独立于特定数据类型的编程技术。在C++中，模板允许编写一次代码，然后将其应用于不同的数据类型。

在C语言标准库中，由于C语言没有直接的泛型编程支持，因此C程序员通过一些巧妙的编程技巧，实现类似的效果。例如C语言标准库中很多函数为了能够支持不同的数据类型，就被设计成了“万能函数”（或者称为通用函数）。

通用函数常常会使用void*作为函数参数，还有通过向函数传递函数指针或宏函数的方式。考虑如下情景，我们需要对一个无序数组进行排序，我们可以使用任何一种排序算法实现此事，也可以使用标准库函数 qsort()来完成此事，但无论是使用自己编写的排序函数还是使用qsort，我们都需要去明确一件事：待排序数组的元素的数据类型到底是什么，这影响着我们的编写的处理函数的参数类型，在这种思路下我们编写的处理函数和需要处理的数据类型高度绑定，我们需要编写一个sort_int函数去处理int类型数组的排序，需要编写一个sort_float函数去处理float类型数组的排序，从算法的层面讲，sort_int和sort_float本质上是没有任何区别的：我们需要遍历整个容器，然后根据不同排序算法的原理对我们遍历的元素进行交换，仅此而已。重复造轮子的过程是没有必要的，c语言通过 void* 指针、函数指针、宏来实现类似的泛型思想。

基于上述问题，C++通过模板技术来支持泛型编程，模板允许函数或类在编译时生成针对特定类型的代码，从而实现类型无关的通用编程。

模板的主要优点是提供了类型安全、可重用的代码，同时避免了运行时的类型检查和转换。我们上述的Array容器使用了模板技术，我们在编写Array的一些成员方法时，无需考虑元素类型，仅考虑算法如何操作该数据类型。

迭代器

模板使得在编写代码时，编写类型无关的代码成为可能。这是泛化（generics）思想的一种体现,模板实现了类型泛化的编程思想，在定义函数、类或数据结构时使用未指定具体类型的参数或成员。编写更加通用和灵活的代码，代码可以独立于具体的数据类型操作。

迭代器（iterators）与泛化有关联。迭代器是一种泛化的概念，它允许在不暴露底层数据结构细节的情况下，对容器中的元素进行遍历和访问。

迭代器提供了一种统一的方式来访问容器中的元素，不论容器中存储的是什么类型的数据。因此，迭代器可以看作是一种容器泛化的体现，它允许编写可以操作各种容器的通用算法和代码。

迭代器的特性

如果 it 是一个迭代器，那么 it 应该支持以下基础操作：

  1. 迭代器应该支持解引用操作，以便可以通过解引用操作访问到迭代器引用的值，即迭代器应该支持 * 运算符。我们可以通过 *it 获取迭代器所指向的数据。

  2. 能够使用迭代器遍历容器中的所有元素，即至少应该对递增运算符进行定义，即可以使用 it++ 或 ++it 来向后遍历一个下一个数据。

  3. 相同类型的数据容器的迭代器应该是能够相互赋值的，即如果存在两个迭代器 it1、it2,那么 it1=it2 赋值操作是有意义的。

  4. 应该对迭代器的比较操作进行定义，即我们可以通过 it1==it2 或 it1!=it2 的返回结果判断两个迭代器是否指向同一个元素或不同元素。

标准容器库（STL）遵循上述原则，每个数据容器定义了相应的迭代器类型，迭代器可能是指针，也可能是对象，但不管迭代器是如何实现的，迭代器都提供所需的操作。每个容器类都有一个超尾标记，当迭代器递增到超越容器的最后一个值后，这个值被赋给迭代器。

使用容器类时，迭代器提供了统一的接口，所有的迭代器类都提供了一个begin()方法，返回指向容器第一个元素的迭代器，end()方法返回一个指向超尾的迭代器，begin()和end()确定了迭代的区间，迭代器重载了递增运算符，因此可以从begin()递增至end()来遍历整个数据容器，再使用*运算符进行元素访问。
